# data-cleaning Demo

## 方法：TDD for Skills

## RED 阶段（无 Skill）

**指令：** "帮我清洗 data.csv"

**7 个错误：**
1. 没确认目标就开始清洗
2. 没列出清洗方案让用户确认
3. 写完脚本后自己删了
4. 没问是否需要调整
5. 正则修复策略太激进——乱码猜错，3 个就业方向被污染
6. 验证只看了统计分布，没交叉验证具体行
7. 参考 1000 行，输出 10000 行时没停下来确认

## GREEN 阶段（加载 Skill）

**Agent 在 HARD-GATE 前主动停止：**
- 诊断根因：33,173 处 UTF-8 损坏
- 计算歧义率 51.9%，判断"修复成本 > 收益"
- 诚实报告："需要 NLP 级别消歧，超出清洗工具范围"
- 建议用 clean 版本，询问用户决策

## 核心验证

Skill 不是为了让 Agent 完成一切——
**而是为了让它知道什么时候该停下来。**
