# 报告生成

## Overview

报告不是把中间产物堆在一起——是结构化的可交付文档。
**核心原则：生成前列清单，生成后有校验。**

<HARD-GATE>
禁止以下行为：
- 禁止在用户确认章节清单之前生成报告
- 禁止把原始脚本输出直接当报告正文
- 禁止报告缺元信息（标题、时间、数据来源、Skill 列表）
</HARD-GATE>

## Checklist

按顺序完成：

### 1. 确认格式

列出可选格式让用户选：

```
A) Markdown（默认，可在 GitHub/编辑器渲染）
B) HTML
C) PDF
D) Word (.docx)
E) 纯文本
```

用户选择后进入下一步。

### 2. 列出章节清单

报告结构（默认包含以下章节，可增减）：

```
1. 封面
   - 报告标题
   - 生成时间
   - 数据来源（文件名 + 样本量）
   - 使用的 Skill 列表

2. 目录
   - 含一级和二级标题
   - 带编号（1 / 1.1 / 1.2）
   - Markdown 格式用锚点链接

3. 数据概况
   - 基本信息（行列数、字段说明）
   - 数据质量（缺失率、异常值）

4. 分析结果（每项假设/每个分析方向一个子章节）
   - 分析问题 / 假设
   - 方法
   - 结果（图表 + 文字解读，图文捆绑）
   - 结论

5. 综合结论
   - 所有发现的汇总表
   - 最重要的 3 条结论

6. 局限性
   - 样本局限
   - 方法局限
   - 未覆盖的分析角度

7. 附录
   - 完整文件路径清单
   - 数据来源说明
```

发给用户："这是计划生成的报告结构，需要增删哪些章节？"
确认后进入下一步。

### 3. 收集分析产出

从已完成的分析步骤中收集产出。**数据来源优先级：**

| 信息类型 | 优先来源 |
|---------|---------|
| 统计检验结果（p值、效应量） | statistical-analysis |
| 图表 + 趋势描述 | data-visualization |
| 数据质量报告 | data-cleaning |
| 数据加载信息 | data-loading |

**规则：**
- 同一个数值出现在多个来源中 → 以 statistical-analysis 的值为准
- statistical-analysis 没有覆盖的数值 → 标注"未经统计检验"
- 不重新分析——只收集已有产出。

### 4. 生成报告

按确认的章节结构生成。排版要求：

| 要求 | 说明 |
|------|------|
| 图表捆绑 | 每张图紧跟其解读文字，中间不隔其他内容 |
| 表格有表头 | 表号 + 表名（如"表 1：各就业方向 GPA"） |
| 图表有图注 | 图号 + 图名 + 数据来源 |
| 目录可跳转 | Markdown 报告使用 `[章节名](#锚点)` |
| 元信息完整 | 标题、时间、数据源、Skill 列表必须在封面 |

### 5. 生成后校验

对照章节清单逐条检查：

- 每个章节都存在？
- 每张图都有对应的文字解读？
- 目录中的链接都能正常跳转？
- 元信息（时间、来源、Skill）完整？

缺任意一项 → 补充后重新校验。

### 6. 清理中间产物

- 最终报告 → `reports/` 目录
- 图表 → `charts/` 目录
- 中间脚本/临时文件 → 列表展示，问用户是否保留
- 不保留的 → 删除

### 7. 输出文件清单

报告生成完成后，输出完整文件清单：

```
📄 报告 → reports/分析报告.md
📊 图表 → charts/01_xxx.png
📊 图表 → charts/02_xxx.png
📜 脚本 → scripts/analysis.py（已保留）
📂 数据 → data/clean_output.csv
```

### 8. 询问反馈

- 问："报告是否满足需求？有没有需要调整的章节？"
- 需要调整 → 回到第 2 步，修改章节清单
- 满意 → 结束

## 常见错误

| 错误 | 正确做法 |
|------|---------|
| 不列章节清单直接生成 | 先列清单让用户确认 |
| 把脚本输出当报告正文 | 只收集结论，不复制代码输出 |
| 图文分离（所有图堆一起） | 图+解读捆绑，一节一节来 |
| 缺元信息 | 封面必须有标题、时间、来源、Skill |
| 目录无链接 | Markdown 锚点让目录可点击 |
| 不清理中间文件 | 问用户，该删的删 |

## Red Flags

| 想法 | 正确做法 |
|------|---------|
| "直接输出就行，不用列章节" | Checklist 第 2 步：列清单让用户确认 |
| "所有图放一起比较方便" | 图+解读捆绑，按章节分 |
| "中间文件留着吧，无所谓" | 列出清单，问用户要不要删 |
| "报告写完了，肯定没问题" | Checklist 第 5 步：逐条校验 |
