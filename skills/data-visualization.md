# 数据可视化

## Overview

可视化不是画图——是回答问题。先知道要回答什么，再选图。
**核心原则：没有确认目标之前，不碰数据。**

<HARD-GATE>
禁止以下行为，直到对应条件满足：
- 禁止加载/分析数据，直到用户确认分析目标
- 禁止自动删除或填充缺失值——只能报告，让用户决定
- 禁止凭感觉选图表——必须查 Chart Selection Guide
- 禁止输出未加标题、轴标签、图例的图表
</HARD-GATE>

## Checklist

按顺序完成，每步一个 TODO：

### 1. 确认分析目标

- 反问用户："你想从这份数据里回答什么问题？"
- 如果用户说不清楚，列出 3 个方向让用户选：
  "我可以帮你做：A)趋势分析 B)对比/排名分析 C)分布分析，你更想看哪个？"
- 用户确认后，复述一遍："所以你想知道 [X] 和 [Y] 的关系，对吗？"
- 确认无误后进入下一步

### 2. 检查数据质量

- 输出数据概况：行数、列数、每列类型、缺失值数量及比例
- 如有缺失值 → 报告，不自动处理：
  "X 列有 Y% 缺失值，你希望：A)删除这些行 B)用均值/众数填充 C)先跳过？"
- 如有明显异常值（如 GPA=0 或超出合理范围）→ 报告并等待确认
- 生成一张测试图，确认中文/特殊字符能正常显示
- 如果字体不支持 → 报告用户，改用英文标签或下载字体

### 3. 选择图表方案

- 查阅下方 Chart Selection Guide，根据数据特征匹配图表类型
- 不允许"觉得这个图好看就选"——必须对号入座
- 列出图表清单发给用户确认，每条格式：
  "• [图表类型] — [分析内容]（因为 [数据特征]，所以选这个图）"
- **发送清单后立即停止，等待用户回复**
- 用户回复"可以"/"确认"/"没问题"后，才进入第 4 步

<HARD-GATE>
禁止在用户确认图表清单之前生成任何图表。清单发送后，下一步动作只能是等待。
</HARD-GATE>

### 4. 生成图表

- 每张图必须有：标题、X轴标签、Y轴标签
- 标题格式："[分析内容]（数据来源：[文件名]）"
- 颜色方案：避免 Jet/Rainbow，使用 viridis/plasma 等可感知均匀的配色
- 保存为 PNG，150 DPI 以上
- 每个文件存到 charts/ 目录

### 5. 添加文字解读

每张图附带 3-4 句描述：
- ① 图中最显著的一个模式或发现
- ② 与整体均值/基准的对比
- ③ 一个可能的原因（必须用"可能"）

### 6. 询问用户反馈

- 列出已生成的所有文件及路径
- 问："这些分析是否满足你的需求？有没有想补充的角度或追加的分析？"
- 用户提出追加 → 回到第1步，针对新需求重复流程
- 用户满意 → 结束

## Chart Selection Guide

| 数据特征 | 问题类型 | 推荐图表 |
|---------|---------|---------|
| 时间序列 + 数值 | 看趋势 | 折线图 |
| 分类 + 数值 | 看排名 | 横向柱状图 |
| 类别 ≤5 + 看占比 | 看组成 | 饼图 |
| 类别 >5 + 看占比 | 看组成 | 堆叠柱状图 |
| 单变量数值 | 看分布 | 直方图 + KDE |
| 分类 + 数值 + 看分布 | 看组间差异 | 箱线图 |
| 两个数值变量 | 看关系 | 散点图 |
| 多个数值变量 | 看相关性 | 热力图 |
| 分类 × 分类 | 看交叉关系 | 堆叠柱状图 / 热力图 |
| 地理数据 | 看空间分布 | 地图 / 气泡图 |

## 图表规范

- 必须项：标题、X轴标签、Y轴标签
- 配色：不使用 Jet/Rainbow，使用 viridis / plasma / cividis / magma
- 字体：确保支持数据中的语言
- 分辨率：≥150 DPI
- 图例：放在不影响数据区域的角落
- 条形图 + 数值标注：值直接标在条上或条末端

## Red Flags

如果出现以下想法，立即停下，回 Checklist：

| 想法 | 正确做法 |
|------|---------|
| "数据看起来挺干净，直接画吧" | 回到 Checklist 第 1 步：你还没确认目标 |
| "缺几个值无所谓，删了就行" | 报告用户，让用户决定 |
| "饼图最常用，就它了" | 查 Chart Selection Guide |
| "用户应该能看懂这图" | 确认标题、标签、图例齐全 |
| "分析完就行了，不用问" | Checklist 第 6 步：必须询问反馈 |

## 常见错误

| 错误 | 正确做法 |
|------|---------|
| 不做 EDA 直接画图 | 先观察数据概况（shape、类型、缺失值） |
| 饼图塞 10+ 个分类 | 改用横向柱状图 |
| 不处理编码问题 | 生成测试图确认中文正常 |
| 图表无标题/无标签 | 标题 + X标签 + Y标签 是最低要求 |
| 选错图表类型 | 查 Chart Selection Guide，对号入座 |
| 自动删除缺失数据 | 报告用户，等待确认 |
