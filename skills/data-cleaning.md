# 数据清洗

## Overview

清洗不是改数据——是诊断 + 报告 + 让用户决定。
**核心原则：猜错的代价 > 留空的代价。**

<HARD-GATE>
禁止以下行为：
- 禁止在用户确认清洗方案之前执行任何清洗操作
- 禁止猜测乱码/损坏字段的修复值——只能标记，不能猜
- 禁止自动删除含缺失值的行——只能报告，让用户选
- 禁止在检测到异常行为后继续执行——必须停止并报告
</HARD-GATE>

## Checklist

按顺序完成，每步一个 TODO：

### 1. 确认目标数据

- 输出数据概况：文件名、行数、列数、前 3 行样本
- 问："确认清洗这份数据吗？"
- 用户说不是 → 让用户提供正确路径，回到上一步
- 如果用户提供了参考文件（干净的标杆数据），记录下来——后续所有清洗都要对齐它

### 2. 探查数据问题

输出一份"数据诊断报告"：

- 编码问题：检测文件编码，标记损坏字段
- 列结构：有没有空列？有没有列合并/分裂？
- 缺失值：每列缺失数量和比例
- 异常值：数值超出合理范围的记录
- 重复行：完全重复或近似重复
- 格式问题：日期格式、数字类型、布尔值格式

### 3. 列出清洗方案

每条格式：`"问题：[具体问题] → 方案：[操作] → 影响：[多少行]"`
最后加一行：`"清洗后预计：X 行 × Y 列"`

发送后**立即停止，等待用户确认。** 用户说"可以"/"确认"后才进入第 4 步。

### 4. 执行清洗

**修复策略：**

| 问题类型 | 操作 |
|---------|------|
| 空列/全重复列 | 直接删除 |
| 编码损坏（可确定修复） | 用字典/规则自动修复 |
| 编码损坏（模糊匹配） | 列出 2-3 个候选值，让用户选 |
| 编码损坏（完全无法判断） | 标记为 ⚠️，保留原始值或填 NaN |
| 缺失值 | 报告比例，让用户选：删行/填均值/填众数/跳过 |
| 异常值 | 报告具体值，让用户确认是否修正 |
| 重复行 | 报告数量，让用户选：删/保留第一条/保留最后一条 |
| 类型错误 | 自动转换，失败的标记出来 |

**红线：不要猜。不确定就标记。**

### 5. 验证清洗结果

四层验证，任一层不通过 → 报告用户：

- ① 值域验证：每个分类列的值是否在合法集合内
- ② 统计验证：数值列的均值/分布是否与参考一致
- ③ 抽样验证：随机抽 10 行，逐字段对比原始数据
- ④ 边界验证：行数、列数、缺失率是否在合理范围

### 6. 异常行为检测

清洗完成后，检查以下 5 条——任一条触发 → 停止，报告用户：

- □ 输出行数与预期差距 > 20%
- □ 某个分类值完全消失（如参考中有"创业者"，输出为 0）
- □ 出现参考文件中不存在的分类值
- □ 数值列出现超出合理范围的值（如 GPA > 4.0）
- □ 缺失率 > 10%

### 7. 询问反馈

- 问："清洗结果是否满足需求？有没有字段需要重新处理？"
- 需要调整 → 回到第 3 步，只处理要调的部分
- 满意 → 进入下一步

### 8. 收尾

- 输出文件保存为 `clean_output.csv`
- 问："是否保留清洗脚本？"（用户决定删/留）
- 结束

## 常见错误

| 错误 | 正确做法 |
|------|---------|
| 没确认数据就开始洗 | 先输出概况，让用户确认 |
| 直接用正则猜测乱码修复 | 标记 ⚠️，让用户决定 |
| 自动删掉带缺失值的行 | 报告比例，让用户选方案 |
| 验证只看统计分布 | 加抽样对比 + 值域检查 |
| 10 倍于参考文件不报告 | 触发异常检测，立即停止 |
| 洗完后直接删脚本 | 问用户要不要保留 |

## Red Flags

| 想法 | 正确做法 |
|------|---------|
| "这个乱码我猜应该是 X" | 不猜。标记为 ⚠️。 |
| "少几行无所谓" | 报告具体影响，让用户判断 |
| "直接全自动清洗" | Checklist 第 3 步：先列方案，等确认 |
| "验证过了，应该没问题" | 四层验证全过才能说没问题 |
